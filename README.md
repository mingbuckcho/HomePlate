HomePlate : 대규모 트래픽에도 무너지지 않는 고가용성 야구 예매 시스템

눈에 보이지 않는 인프라의 안전이 서비스의 신뢰를 만듭니다.
단 한 번의 실수도 허용되지 않는 세무/의료 현장에서 체득한 무결성에 대한 집착을, HomePlate의 클라우드 인프라와 CI/CD 파이프라인에 녹여냈습니다.


<br/>


- 프로젝트명: HomePlate (통합 야구장 예매 플랫폼)
- 개발 기간: 2026.01.23 ~ 2026.03.20 (2개월)
- 참여 인원: 5명
- 나의 역할: CI/CD 파이프라인 설계, 인프라 구축 및 최적화, 형상 관리 총괄

<br/>
<br/>

Tech Stack
- Cloud & Infra: `AWS (EKS, EC2, VPC, ALB, S3, RDS)`
- CI/CD: `Jenkins`, `ArgoCD`, `Kaniko`, `SonarQube`
- Observability: `Prometheus`, `Grafana`, `Loki`, `Tempo`
- Data & Queue: `MySQL`, `Redis`, `Kafka`

<br/>
<br/>

핵심 성과

1. Shift-Left Security
단순히 코드를 배포하는 것에 그치지 않고, 보안 취약점이 섞인 코드가 라이브 환경에 올라가는 것을 원천 차단했습니다.
- SonarQube Quality Gate: 보안 등급 A, 취약점 0개 기준 적용
- Jenkins 파이프라인과 연동하여 기준 미달 시 배포 즉각 중단 프로세스 구축
- 하드코딩된 비밀번호 및 민감 정보 노출 리스크 사전 차단율 100% 달성

2. 비용과 성능의 최적화
EKS 환경에서 무작정 서버 사양을 높이는 대신, 실제 자원 사용량을 분석하여 병목을 해결했습니다.
- 문제: 배포 시 CPU 예약값 초과로 인한 Pod Pending 현상 발생
- 해결: 서비스별 실제 메트릭을 분석하여 CPU Requests를 150m에서 50m으로 최적화
- 결과: 노드 풀 활용도 극대화로 배포 성공률 100% 달성 및 인프라 운영 비용 30% 절감

3. GitOps 기반의 안전한 롤백
수동 배포에서 오는 휴먼 에러를 가장 경계했습니다.
- ArgoCD를 도입하여 쿠버네티스 클러스터와 Git 저장소 상태를 실시간 동기화
- 장애 발생 시 버튼 한 번으로 즉각적인 롤백이 가능한 선언적 배포 환경 완성
